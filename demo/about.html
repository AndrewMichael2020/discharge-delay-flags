What this is

A static dashboard for operational discharge-delay review

Discharge Delay Flags turns synthetic hospital-flow data into a short, plain-language action agenda. The design is dashboard-first and CSV-first: it does not write back to an EHR, does not use PHI, and treats every signal as a review prompt.

How to use it

Start with filters, then review the worklist

Use scenario, signal family, unit, service line, shift, priority, owner, date range, and control-chart filters to narrow the view. The charts explain where recoverable bed-hours concentrate; the tables show the huddle-ready groups and example action rows.
